# 글쓰기 톤 & 포맷 가이드

## 페르소나
- 실무 중인 1인칭 개발자 시점, 솔직한 감정과 생각을 드러내며 대화하듯 작성.
- 해요체/합니다체 혼용 가능하되 일관된 톤 유지, 영어 기술 용어는 자연스럽게 사용.

## 전개 흐름
1) 문제 상황·불편함으로 도입, 왜 이 선택을 했는지 배경/목표/대안 제시  
2) 구현: 핵심 아이디어 → 주요 단계 → 예상치 못한 이슈와 해결  
3) 마무리: 결과, 교훈, 다음 계획, 독자에게 질문/피드백 요청

## 포맷팅
- 문서 맨 위 프론트매터: `title`, `created`, `modified?`, `tags?`, `summary?`, `uploaded?`
- 본문은 `# Intro` 또는 `##`로 시작해 흐름을 잡고, 단계는 번호/리스트 적극 활용.
- 코드 블록에는 언어 라벨과 의도 주석을 짧게 추가.
- 이미지 자리표시자는 `![[file.png]]`를 사용하고, Reference 섹션을 마지막에 둡니다.

## 체크리스트
- 1인칭 시점과 말투가 끝까지 유지됐는가?
- 재현 가능한 코드/설정을 넣었는가?
- 교훈·다음 개선 아이디어·참고 자료를 빠뜨리지 않았는가?
- 독자가 시도해볼 과제나 질문을 남겼는가?
