# 블로그 카테고리 가이드 (Set A 기준)

폴더명은 영어 Title Case, 설명은 한국어. 글의 주제/목적에 따라 분류하고, 역할(FE/BE/모바일)은 태그로 보조한다.

- **Product Builds**: 특정 제품/기능을 0→1로 만든 여정과 의사결정. 예) 지도 설계, 위치 기반 서비스, 인증 문서 자동 생성.
- **Architecture Patterns**: 설계, 도메인 분해, 재사용 가능한 패턴/구조. 예) Feature-Sliced Design, 리디렉션/훅 패턴.
- **Devops Automation**: CI/CD, 배포, 모노레포, 로깅, 환경 설정/최적화. 예) GitHub Actions, CloudWatch 파이프라인, 로컬 HTTPS+LB.
- **Security Reliability**: 인증/암호화/토큰/접근 제어, 안정성 확보. 예) AES/KMS 업로드, Supabase 쿠키 토큰, DI 중복 탐지.
- **Data Docs**: 데이터/문서/OCR/지도/인덱싱 처리 파이프라인. 예) CLOVA OCR, PDF 분할, 대용량 마커 클러스터링.
- **Troubleshooting Playbook**: 에러/버그/장애 대응 사례와 체크리스트. 예) 기존 Troubleshooter's Corner 전반.
